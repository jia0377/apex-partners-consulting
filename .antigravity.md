You are working on harness-company, a Next.js 15 company introduction / portfolio website template with an admin panel.

## Stack
- Next.js 15 (App Router), React 19, TypeScript (strict)
- Tailwind CSS v4, Framer Motion, Supabase, Lucide React, Pretendard font

## Architecture
- All site content is centralized in `src/lib/data.ts`
- Colors/theme defined in `tailwind.config.ts`
- Supabase handles contact form storage and admin authentication
- Admin panel at `/admin` is protected by `ProtectedRoute` component
- `useAuth` hook in `src/hooks/useAuth.ts` manages session state

## Key Files
- `src/lib/data.ts` — Single source of truth for ALL site text and configuration
- `tailwind.config.ts` — Brand colors (mint/teal) and font family
- `src/app/layout.tsx` — Root layout with Pretendard font and global nav/footer
- `src/components/ui/AnimatedSection.tsx` — Intersection observer + Framer Motion wrapper
- `src/lib/supabase.ts` — Supabase client initialization
- `src/hooks/useAuth.ts` — Auth state management hook

## Component Organization
- `src/components/ui/` — Reusable primitives (Button, Card, AnimatedSection, PageHeader)
- `src/components/sections/` — Home page sections (Hero, About, Services, Portfolio, Testimonials, Contact)
- `src/components/layout/` — Navbar (with mobile menu), Footer
- `src/components/admin/` — Admin dashboard (AdminLayout, ContactTable, ProtectedRoute)

## Customization Guide
| What to change | Where to edit |
|---|---|
| All text/content | `src/lib/data.ts` |
| Brand colors | `tailwind.config.ts` → `colors.brand` |
| Font | `src/app/layout.tsx` + `public/fonts/` |
| Navigation | `src/lib/data.ts` → `navLinks` |
| Contact form fields | `src/components/sections/Contact.tsx` |
| Admin table columns | `src/components/admin/ContactTable.tsx` |
| Animation settings | `src/components/ui/AnimatedSection.tsx` |

## Conventions
- Named exports for all components
- `'use client'` directive on interactive components
- `cn()` utility for conditional Tailwind classes
- Lucide React icons imported individually by name
- Comments mark customization points throughout the codebase

## Deployment: GitHub → Vercel
1. GitHub push → Vercel 자동 재배포 (main 브랜치)
2. Vercel 환경변수: `.env.example` 키 전부 입력
3. 환경변수 수정: Vercel 대시보드 → Settings → Environment Variables

## DB 선택: Supabase vs Firebase
이 템플릿은 Supabase 사용. 구조화된 데이터(contacts 테이블, admin auth)에 최적.
Firebase 대안: Firestore + Firebase Auth로 대체 가능하나, SQL 기반 데이터는 Supabase가 적합.

## SEO 설정
- `src/app/sitemap.ts` — sitemap.xml 자동 생성
- `src/app/robots.ts` — robots.txt 자동 생성 (admin 경로 차단 포함)
- `.env.local`에 `GOOGLE_SITE_VERIFICATION`, `NAVER_SITE_VERIFICATION` 입력
- `public/og-image.png` (1200×630px) 추가 시 SNS 공유 미리보기 활성화

## 도메인 연결 (가비아 → Cloudflare → Vercel)
1. 가비아에서 도메인 구매
2. Cloudflare에 도메인 등록 → 네임서버 복사
3. 가비아 네임서버를 Cloudflare 네임서버로 변경
4. Vercel Settings → Domains에 도메인 추가
5. Cloudflare DNS에 CNAME 레코드 추가 (Proxy: DNS only)
6. `NEXT_PUBLIC_SITE_URL` 실제 도메인으로 변경 후 재배포
